# 管理后台说明

后台页面保留在小程序内，但普通用户界面不展示后台入口。管理员可通过开发者工具或内部路径进入 `/pages/admin/login/index`。

## 管理员登录

上线前必须在 `admin_users` 集合中手动写入管理员 openid，且 `status` 为 `active`。系统不会自动把首次访问后台的用户初始化为管理员。

示例：

```json
{
  "openid": "管理员微信 openid",
  "role": "super_admin",
  "status": "active",
  "created_at": "云数据库日期",
  "updated_at": "云数据库日期",
  "deleted_at": null
}
```

## 数据概览

展示总用户数、今日新增用户数、今日饮食记录数、今日运动记录数、今日体重记录数、待处理反馈数。后台第一版不展示用户详细健康数据。

## 用户管理

支持搜索、查看账号状态、禁用用户、恢复用户。禁用用户再次登录会被拒绝，受限云函数也会因用户状态非 `active` 拒绝操作。

## 食材维护

支持食材分类和平台食材的新增、编辑、上架、下架、删除。下架和删除操作均需要二次确认。

## 食谱维护

后台“食谱管理”支持新增、编辑、上架、下架和设置推荐；“标签管理”支持新增、编辑、停用和删除标签。

食谱配方只能选择已上架的平台食材，保存时由云函数重新读取食材营养并计算每份热量、蛋白质、碳水和脂肪。新食谱默认下架，确认内容完整后再开启“上架展示”。下架食谱不会出现在用户端，也不能继续加入饮食或一日计划。

## 动作与训练计划维护

后台“动作管理”支持新增、编辑、上架和下架动作；动作名称、分类、目标肌群、难度、器械、步骤、常见错误和注意事项为必填内容，封面和视频地址可选。

后台“训练计划管理”支持新增、编辑、上架和下架计划。计划上架时，其中每个动作都必须处于上架状态；动作或计划下架后，小程序端列表与详情立即不可见，使用该内容的当前计划也会提示用户重新选择。

## 健康文章维护

后台“文章管理”支持按标题搜索，新增或编辑饮食知识、训练知识、记录技巧和常见问题文章，并可上架、下架、设置或取消首页推荐。新文章默认下架，确认标题、摘要、正文和可选封面后再上架；下架文章不会出现在用户端列表、详情和首页推荐中。

## 食材纠错处理

后台“食材纠错”支持按待处理、处理中、已采纳、未采纳筛选，查看用户说明、建议值、提交时食材快照和当前食材数据。管理员可填写处理备注；只有状态设为“已采纳”时才能勾选“同步修改食材”，保存后平台食材和纠错结果在同一事务中更新。后台不展示提交者的健康档案或其他私有记录。

## 反馈处理

支持查看反馈内容、修改处理状态、填写管理员回复。状态包括：待处理、处理中、已处理、已关闭。
