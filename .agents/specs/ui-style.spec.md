# UI Style Spec: 健身饮食小程序 MVP

## 目标

统一 Phase 1-4 的界面风格，让小程序呈现清爽、健康、专业的记录工具气质。界面优先服务高频记录、快速扫描和明确反馈，不做营销型首页，不做复杂装饰。

## 设计原则

- 工具优先：首屏优先显示今日热量、已摄入、剩余热量、运动消耗和营养素进度。
- 信息清晰：用稳定的卡片、列表和表单结构承载数据，避免过度视觉包装。
- 轻量健康：主色使用健康绿色，辅以蓝色、橙色表达营养素和提醒，不使用大面积渐变、装饰球、营销插画。
- 可连续记录：常用入口放在首页和底部 Tab，空状态必须给下一步入口。
- 状态完整：加载、空状态、失败、成功反馈要可见，用户提交失败时保留已输入内容。

## 颜色

全局颜色变量定义在 `miniprogram/app.wxss`，后续页面优先使用变量，不直接散落新颜色。

- `--color-primary: #16a05d`：主按钮、关键操作、健康正向状态。
- `--color-primary-dark: #0e7a46`：重要数字、强调文字。
- `--color-accent-blue: #2f80ed`：碳水进度、信息状态。
- `--color-warning: #f5a524`：脂肪进度、提醒状态。
- `--color-danger: #d93025`：删除、失败、危险操作。
- `--color-text: #1f2d26`：主文本。
- `--color-muted: #7a8580`：辅助文本、说明。
- `--color-border: #e4ebe7`：分割线、输入框边框。
- `--color-bg: #f5f7f6`：页面背景。
- `--color-card: #ffffff`：卡片与表单容器背景。

## 字体层级

- 页面标题 `.title`：40rpx，700，用于页面主标题。
- 区块标题 `.section-title`：32rpx，700，用于卡片或模块标题。
- 列表标题 `.list-title`：30rpx，600，用于食物名、记录名、菜单项。
- 正文 `.body-text`：28rpx，400，用于普通内容。
- 辅助说明 `.subtitle` / `.muted-text`：24-26rpx，颜色使用 `--color-muted`。
- 数据大字 `.metric-value`：56-72rpx，700，用于首页剩余热量等核心指标。

不要使用随视口变化的字体大小，不使用负字距。

## 布局与间距

- 页面容器 `.page`：左右 28rpx，顶部 32rpx，底部 48rpx。
- 页面模块 `.section`：模块间距 28rpx。
- 小间距：8rpx / 12rpx。
- 中间距：16rpx / 20rpx / 24rpx。
- 大间距：28rpx / 32rpx。
- 固定格式元素如仪表盘、按钮、列表行、表单项要有稳定高度或最小高度，避免加载和动态文字导致布局跳动。

## 卡片样式

- 基础卡片 `.card`：
  - 白底。
  - 1rpx 边框 `--color-border`。
  - 12rpx 圆角。
  - 28rpx 内边距。
- 卡片只用于承载一组具体内容、列表项、表单、结果模块。
- 不要卡片套卡片。
- 页面大区块不要做漂浮营销卡片。

## 按钮样式

- 主按钮 `.primary-button`：用于保存、登录、添加等主操作。绿色实底，白字，最小高度 88rpx。
- 次按钮 `.secondary-button`：用于次要导航和非破坏操作。白底、边框、深色文字。
- 小按钮 `.small-button`：用于列表行内操作，如添加、更换、复制昨日。
- 危险按钮 `.danger-button`：用于删除。文字或边框使用 `--color-danger`，避免大面积红色。
- 按钮文字必须完整可见，不允许挤压或溢出。

## 表单样式

- 表单容器使用 `.form-card`。
- 表单项使用 `.field`，label 使用 `.label`。
- 输入框、选择器使用 `.input` / `.picker-value`：
  - 高度 78rpx。
  - 20rpx 横向内边距。
  - 1rpx 边框。
  - 12rpx 圆角。
  - 白底。
- 必填项在 label 后标 `*`。
- 所有表单必须保留前端校验和服务端校验。
- 提交失败时保留用户已输入内容。

## 列表样式

- 列表容器使用 `.list`，垂直间距 16rpx。
- 列表项使用 `.list-card` 或业务语义类继承 `.card`。
- 列表项结构建议：
  - 左侧主信息：名称、摘要、营养数据。
  - 右侧操作：添加、删除、编辑入口。
- 食物和饮食记录列表要优先显示名称、重量、热量，再显示蛋白质、碳水、脂肪。

## 空状态

- 使用 `.empty`。
- 文案说明当前没有什么数据。
- 必须提供下一步入口，比如“添加饮食”“新建自定义食物”“填写健康档案”。
- 空状态不使用插画作为必要信息来源，第一版保持轻量。

## 加载状态

- 异步操作使用统一 `showLoading/hideLoading`。
- 页面局部加载可使用 `.loading-state`：
  - 白底或透明背景。
  - 文案不超过一行。
  - 不遮挡已输入表单内容，除非是提交中的全局 loading。

## 首页仪表盘规范

首页必须优先展示：

- 今日推荐摄入热量。
- 今日已摄入热量。
- 今日剩余热量。
- 今日运动消耗，Phase 2 可显示 0。
- 蛋白质、碳水、脂肪进度。
- 早餐、午餐、晚餐、加餐入口。
- 今日饮食打卡状态。

视觉层级：

1. 剩余热量作为最大数字。
2. 推荐、已摄入、运动消耗为同一行三列。
3. 三大营养素使用进度条或紧凑数据行。
4. 常用操作按钮放在仪表盘下方。

## 页面约束

- Phase 1-4 新页面必须使用本规范的颜色、字体、卡片、按钮、表单、列表、空状态。
- 不引入会员、支付、AI 拍照识别、扫码识别、社区等 MVP 外功能。
- 不做营销型首页、复杂视觉装饰、大面积渐变、装饰光斑、无意义插画。
- 保持微信小程序原生实现，优先复用 `app.wxss` 基础类。

## 验收清单

- 页面整体清爽、健康、专业，不像营销落地页。
- 首页核心指标一屏内可见。
- 页面文字不重叠、不溢出按钮或卡片。
- 表单、按钮、列表、空状态样式一致。
- 新增/编辑/删除记录后，相关页面状态清晰反馈。
- 不破坏已完成业务功能。
