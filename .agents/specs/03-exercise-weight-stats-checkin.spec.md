# Phase 3 Spec: 运动、体重、数据趋势、打卡

## 目标

补齐运动记录、体重记录、简单数据趋势和连续打卡，让用户看到饮食之外的身体反馈。

## 页面范围

- 添加运动页
- 运动记录页
- 体重记录页
- 数据页
- 打卡状态组件

## 运动记录

字段：

- 日期
- 运动类型
- 运动名称
- 运动时长，分钟
- 运动强度，低 / 中 / 高
- 消耗热量
- 备注，可选

运动类型：

- 力量训练
- 跑步
- 快走
- 骑行
- 游泳
- 跳绳
- 瑜伽
- HIIT
- 其他

规则：

- 运动时长必须大于 0
- 消耗热量可系统估算，也可用户手动修改
- 添加运动后首页运动消耗同步增加
- 删除运动后首页运动消耗同步减少

## 体重记录

字段：

- 日期
- 体重 kg
- 体脂率，可选
- 备注，可选

规则：

- 同一用户同一天只保留一条体重记录
- 当天重复提交时更新原记录
- 添加体重后自动完成体重打卡

## 数据趋势

第一版展示：

- 近 7 天体重趋势
- 近 7 天热量摄入趋势
- 近 7 天运动消耗趋势
- 今日营养素占比

空数据时展示空状态和引导按钮。

## 打卡规则

打卡类型：

- 饮食
- 运动
- 体重

规则：

- 添加任意饮食记录后，自动完成饮食打卡
- 添加任意运动记录后，自动完成运动打卡
- 添加体重记录后，自动完成体重打卡
- 每天完成任意一种打卡，即计入连续打卡
- 连续打卡按自然日计算
- 中断后连续天数归零

## 验收标准

- 可新增、编辑、删除运动记录
- 运动时长为 0 或负数时不能提交
- 首页运动消耗随运动记录变化
- 可新增和修改今日体重
- 同一天体重重复提交时更新原记录
- 数据页可展示近 7 天趋势
- 无数据时展示空状态
- 打卡状态可根据记录自动完成
- 连续打卡天数计算正确
