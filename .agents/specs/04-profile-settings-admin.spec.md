# Phase 4 Spec: 我的页面、设置、反馈、基础后台

## 目标

完成用户个人管理入口和运营后台基础功能，让 MVP 具备可运营、可反馈、可维护的数据能力。

## 小程序页面范围

- 我的页面
- 设置页
- 意见反馈页
- 用户协议页
- 隐私政策页
- 账号注销页

## 我的页面

展示：

- 头像
- 昵称
- 当前目标
- 连续打卡天数
- 健康档案入口
- 目标设置入口
- 我的自定义食物
- 意见反馈
- 用户协议
- 隐私政策
- 设置
- 退出登录
- 注销账号

## 设置页

第一版支持：

- 查看当前登录状态
- 退出登录
- 进入注销账号页

第一版不做：

- 订阅消息提醒
- 手机号绑定
- 多账号绑定

## 意见反馈

反馈类型：

- 功能异常
- 食物数据错误
- 运动记录问题
- 产品建议
- 其他

字段：

- 用户 ID
- 反馈类型
- 反馈内容
- 图片附件，可选
- 联系方式，可选
- 处理状态
- 管理员回复
- 提交时间
- 处理时间

## 账号注销

规则：

- 注销前二次确认
- 注销后用户状态变为注销
- 健康档案、饮食记录、运动记录、体重记录、打卡记录删除或匿名化
- 注销后重新进入需重新登录和建档

## 管理后台范围

后台第一版功能：

- 管理员登录
- 数据概览
- 用户管理
- 食材分类管理
- 食材管理
- 反馈管理

## 数据概览

展示：

- 总用户数
- 今日新增用户数
- 今日饮食记录数
- 今日运动记录数
- 今日体重记录数
- 待处理反馈数

## 用户管理

支持：

- 用户列表
- 按昵称或用户 ID 搜索
- 查看注册时间
- 查看账号状态
- 禁用用户
- 恢复用户

后台第一版不直接展示用户详细健康数据。

## 食材管理

支持：

- 新增食材
- 编辑食材
- 上架食材
- 下架食材
- 删除食材
- 分类管理

## 反馈管理

支持：

- 查看反馈列表
- 查看反馈详情
- 修改处理状态
- 填写管理员回复

处理状态：

- 待处理
- 处理中
- 已处理
- 已关闭

## 验收标准

- 我的页面入口完整
- 用户可提交反馈
- 反馈内容可在后台查看
- 管理员可登录后台
- 管理员可查看基础数据概览
- 管理员可禁用和恢复用户
- 管理员可维护食材分类和食材
- 管理员可处理反馈
- 用户可退出登录
- 用户可申请注销账号
