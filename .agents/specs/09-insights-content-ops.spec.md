# Phase 9 Spec: 数据洞察与内容运营

## 目标

让数据反馈更有解释性，让后台具备基础内容运营能力。Phase 9 不做付费报告，只做免费、轻量、可理解的数据总结。

## 明确不做

- 付费周报
- 付费月报
- 健康报告付费导出
- 医疗诊断
- 会员权益
- 支付

## 页面范围

- 周总结页
- 月总结页
- 饮食洞察页
- 目标进度页
- 健康文章列表页
- 健康文章详情页
- 后台内容管理页
- 食材纠错管理页

## 功能 1: 周总结

内容：

- 本周平均摄入热量
- 本周训练次数
- 本周体重变化
- 饮食记录天数
- 运动记录天数
- 最常吃食物
- 简单建议

规则：

- 免费展示
- 不生成医疗建议
- 数据不足时展示空状态

验收：

- 有数据时能生成周总结
- 无数据时有引导
- 总结数据与原始记录一致

## 功能 2: 月总结

内容：

- 月度体重变化
- 月度平均热量摄入
- 月度运动次数
- 最长连续打卡天数
- 营养素达标情况
- 下月建议

验收：

- 可查看当前月总结
- 可切换历史月份
- 数据不足时提示继续记录

## 功能 3: 饮食洞察

提示规则：

- 热量连续 3 天超过目标 10%，提示关注摄入
- 蛋白质连续 3 天低于目标 80%，提示增加优质蛋白
- 晚餐热量占比连续 3 天超过全天 40%，提示调整餐次结构
- 连续 2 天未记录饮食，提示恢复记录

注意：

- 文案避免医疗化表达
- 提示只作为健康管理建议

验收：

- 满足条件时展示提示
- 不满足条件时不强行提示
- 提示文案温和清晰

## 功能 4: 健康文章

文章类型：

- 饮食知识
- 训练知识
- 记录技巧
- 常见问题

后台支持：

- 新增文章
- 编辑文章
- 上架文章
- 下架文章
- 首页推荐

验收：

- 小程序端可查看文章列表和详情
- 后台下架文章后小程序端不可见

## 功能 5: 食材纠错

规则：

- 用户可对食材数据提交纠错
- 后台可查看纠错内容
- 后台可处理纠错并修改食材
- 处理状态可追踪

验收：

- 用户可提交食材纠错
- 后台可处理纠错
- 纠错处理后食材数据更新

## 数据建议

新增集合：

- `health_articles`
- `food_corrections`

周总结、月总结和饮食洞察直接从用户原始记录实时计算，不持久化 `weekly_summaries`、`monthly_summaries` 或 `insight_rules` 副本，避免编辑、删除记录后出现统计不一致。

## 验收标准

- 周总结、月总结可免费查看
- 饮食洞察规则准确
- 健康文章可维护和展示
- 食材纠错流程可用
- 不出现付费报告、会员、支付入口
