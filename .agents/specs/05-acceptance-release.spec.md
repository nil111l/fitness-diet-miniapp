# Phase 5 Spec: 联调、验收、发布准备

## 目标

完成 MVP 的整体联调、回归测试、问题修复和交付准备，确保可以提交体验版或交付开发成果。

## 联调范围

必须完整跑通：

- 新用户登录建档流程
- 老用户再次进入流程
- 修改健康档案流程
- 添加饮食流程
- 编辑饮食流程
- 删除饮食流程
- 复制昨日饮食流程
- 添加运动流程
- 删除运动流程
- 添加体重流程
- 查看数据趋势流程
- 提交反馈流程
- 管理后台维护食材流程
- 管理后台处理反馈流程

## 测试账号

至少准备：

- 新用户测试账号
- 已建档普通用户
- 有 7 天记录的用户
- 管理员账号
- 被禁用用户

## 回归测试重点

- 首页数据是否准确
- 统计图数据是否准确
- 用户数据是否隔离
- 删除记录后数据是否同步变化
- 登录过期是否能重新登录
- 未登录是否无法访问受限页面
- 禁用用户是否无法继续使用
- 表单异常值是否被拦截
- 空状态是否有引导

## 性能要求

- 首页首屏加载不超过 2 秒
- 食材搜索响应不超过 1 秒
- 列表分页每页 10-20 条
- 图表只加载必要时间范围数据

## 安全要求

- 小程序端不出现 AppSecret
- 小程序端不出现 session_key
- 用户不能读取他人记录
- 管理后台接口需要管理员身份
- 删除、禁用、下架操作需要二次确认

## 交付物

项目交付时应包含：

- 小程序前端代码
- 云函数代码
- 数据库集合与字段说明
- 管理后台代码或入口说明
- 接口说明
- 部署说明
- 测试账号
- 管理员账号
- 已知问题清单
- 后续二期建议清单

## 验收标准

- MVP 核心路径全部跑通
- P0 和 P1 功能无阻塞问题
- 首页、饮食、运动、体重、数据、我的、后台均可正常使用
- 用户隐私和数据隔离符合要求
- 体验版可提交或成果可交付
